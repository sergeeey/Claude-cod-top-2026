## Analyst Observations

### Где скил даёт максимум (eval-reframe, eval-stuck)
- Наивная рамка: with-skill называет её явно ВСЕГДА. Baseline — никогда. Это структурный паттерн, не случайность.
- Кросс-домен: with-skill использует аналогии из медицины, эволюционной психологии, экономики. Baseline остаётся в родном домене.
- One vs many: with-skill заканчивается ОДНИМ действием. Baseline даёт 3 шага всегда.

### Где baseline силён (eval-extract)
- Baseline на "вытащи мысль" дал 75% — почти догнал with-skill (100%).
- Причина: задача "вытащи мысль" менее структурозависима. Claude и без скила умеет формулировать.
- Вывод: скил особенно ценен для reframe/озарение задач, менее критичен для extract.

### Не-дискриминирующее assertion
- "не разводит на 7 пунктов" — оба прошли 2/3 случаев. Слабый дискриминатор.
- Заменить в iteration 2 на: "ответ умещается в 300 слов без потери структуры"

### Общий вердикт
**with_skill > baseline на всех eval.** Дельта +62 pp — скил работает.
Основная ценность: структура INSIGHT-цикла меняет поведение модели системно, а не случайно.
