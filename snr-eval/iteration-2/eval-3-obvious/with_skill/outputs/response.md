# SNR-анализ: три конкурирующие задачи недели

## Проверка "Правила очевидного случая"

Перед расчётом — проверяю, нужна ли формула:

- Один вариант блокирует все остальные? **Нет** — нет красного CI, нет внешнего блокера.
- Задача < 2 мин? **Нет** — все три задачи занимают дни/часы.
- Grep/Bash дал полный ответ? **Нет** — это judgment-call с конкурирующими критериями.

**Вывод:** три задачи конкурируют за один временной слот, нет явного блокера, все "важные" на входе — классический случай, когда формула нужна обязательно. Считаю SNR.

---

## Скоринг по задачам

### Формула
```
Signal = Impact × Compounding × Leverage × Reusability × Confidence
Noise   = Time + Fragmentation + Coordination + Rework + Governance
SNR     = Signal / Noise
```

---

### Задача B: 5 user interviews (слоты только на этой неделе)

| Параметр | Оценка | Обоснование |
|---|---|---|
| **Impact** | 5 | Прямой вход в product direction нового продукта |
| **Compounding** | 5 | Паттерны из интервью будут влиять на дизайн месяцами |
| **Leverage** | 5 | Снимает неопределённость по десяткам следующих решений (feature-scope, positioning, ICP) |
| **Reusability** | 4 | Инсайты станут knowledge asset — цитаты, personas, jobs-to-be-done |
| **Confidence** | 5 | Данные надёжны: живые люди, не опросы |
| **Time** | 3 | ~5 × 45 мин = ~4 часа + нотирование |
| **Fragmentation** | 3 | 5 временных блоков разбивают рабочий день |
| **Coordination** | 4 | 5 внешних людей, риск переноса |
| **Rework** | 1 | Интервью = первичные данные, не переделываются |
| **Governance** | 1 | Никаких разрешений не нужно |

**Signal** = 5 × 5 × 5 × 4 × 5 = **2500**
**Noise** = 3 + 3 + 4 + 1 + 1 = **12**
**SNR** = 2500 / 12 = **208.3 → KEEP**

**Ключевой фактор:** слоты необратимо исчезают в конце недели. Это единственная задача с жёстким, невозобновляемым дедлайном *прямо сейчас*. Leverage = 5 потому что product decisions без customer data — это рабочие гипотезы, не факты.

### Задача A: RAG pipeline для CogniRouter (дедлайн через 2 недели)

| Параметр | Оценка | Обоснование |
|---|---|---|
| **Impact** | 5 | Ключевая фича CogniRouter — прямой impact на продукт |
| **Compounding** | 4 | Pipeline будет использоваться и развиваться дальше |
| **Leverage** | 3 | Снимает только технический долг по этой фиче, не product uncertainty |
| **Reusability** | 4 | Компоненты pipeline переиспользуемы |
| **Confidence** | 4 | Требования понятны, стек знакомый |
| **Time** | 5 | 3 дня концентрированной работы = высокая стоимость |
| **Fragmentation** | 2 | Требует deep work блоками, но их можно спланировать |
| **Coordination** | 1 | Solo-задача |
| **Rework** | 2 | Риск переделки если архитектура не продумана |
| **Governance** | 1 | Нет |

**Signal** = 5 × 4 × 3 × 4 × 4 = **960**
**Noise** = 5 + 2 + 1 + 2 + 1 = **11**
**SNR** = 960 / 11 = **87.3 → KEEP**

**Ключевой фактор:** дедлайн через 2 недели даёт гибкость. 3 дня работы можно запланировать на следующей неделе. Это высокий SNR, но не ограниченный временно́е окном *прямо сейчас*.

### Задача C: технический пост в блог ("желательно к пятнице")

| Параметр | Оценка | Обоснование |
|---|---|---|
| **Impact** | 3 | Маркетинговый вклад, не прямой продуктовый |
| **Compounding** | 2 | Пост устаревает, трафик спадает |
| **Leverage** | 1 | Не снимает неопределённость ни для какого решения |
| **Reusability** | 3 | Контент можно переиспользовать частично |
| **Confidence** | 4 | Понятно что писать |
| **Time** | 4 | Несколько часов на качественный технический текст |
| **Fragmentation** | 3 | Требует непрерывного времени для написания |
| **Coordination** | 2 | Согласование с маркетингом |
| **Rework** | 3 | Правки от редактора вероятны |
| **Governance** | 1 | Нет |

**Signal** = 3 × 2 × 1 × 3 × 4 = **72**
**Noise** = 4 + 3 + 2 + 3 + 1 = **13**
**SNR** = 72 / 13 = **5.5 → SIMPLIFY**

**Ключевой фактор:** дедлайн "гибкий", слово "желательно" = мягкий запрос, не обязательство. Leverage = 1 — пост не снимает неопределённость ни для каких других решений.

## Рекомендация

**Делай первым: B (user interviews).**

Не потому что SNR самый высокий (хотя он и правда 208), а потому что это единственная задача с невозобновляемым окном. Интервью, которые не состоялись — не переносятся, они теряются. Все остальные задачи допускают сдвиг без потери ценности.

**Практически:**
1. Сегодня — подтверди все 5 слотов интервью, проведи максимум из них на этой неделе.
2. В пятницу или в выходные — набросай черновик поста (1-2 часа), сдай маркетингу с пометкой "черновик, требует правки". SNR 5.5 = SIMPLIFY, не ELIMINATE — требование маркетинга реальное, просто не срочное.
3. Следующая неделя — забронируй 3 дня для RAG pipeline. Deep work требует планирования слотов заранее.

**Что не делать:** начинать RAG pipeline сейчас. У него высокий SNR, но нет дедлайна этой недели. Начать и прервать на полпути — хуже, чем отложить и сделать непрерывно.
