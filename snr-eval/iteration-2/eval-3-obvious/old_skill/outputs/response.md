## Скоринг задач

### A. RAG pipeline для CogniRouter

| Signal | Оценка | Обоснование |
|--------|--------|-------------|
| Impact | 4 | Прямо влияет на продуктовый результат |
| Compounding | 5 | Ляжет в основу архитектуры, будет использоваться месяцами |
| Leverage | 4 | Разблокирует следующие итерации продукта |
| Reusability | 4 | Pipeline → переиспользуемый компонент |
| Confidence | 4 | Задача понятна, требования ясны |
| **Signal = 4×5×4×4×4 = 1280** | | |

| Noise | Оценка | Обоснование |
|-------|--------|-------------|
| Time | 4 | 3 дня концентрированной работы |
| Fragmentation | 2 | Глубокая работа, мало переключений |
| Coordination | 1 | Работа индивидуальная |
| Rework | 2 | Умеренный риск — могут поменяться требования |
| Governance | 1 | Нет sensitive permissions |
| **Noise = 4+2+1+2+1 = 10** | | |

**SNR(A) = 1280 / 10 = 128 → KEEP**

## Рекомендация

**Сначала — B (интервью).**

Причина не только в цифрах. Ключевой принцип SNR: срочность — свойство дедлайна, сигнал — свойство причинного воздействия. Но здесь срочность и сигнал совпадают: слоты исчезнут, и это необратимо. RAG pipeline можно сделать через неделю — он никуда не уйдёт. Собеседников не вернёшь.

**Порядок на неделе:**

1. **Пн–Пт:** провести все 5 интервью (4-5 слотов по 45-60 мин + writeup)
2. **Вт–Чт (блоки по 3-4 часа):** параллельно начать RAG pipeline — глубокая работа в свободные окна между интервью
3. **Пятница / следующая неделя:** блог-пост — либо задержать до конца sprint, либо делегировать черновик маркетингу, добавить технические детали самостоятельно

**Для блог-поста:** скажи маркетингу "к пятнице не успею — дам к понедельнику следующей недели". SNR = 3 не стоит того, чтобы жертвовать интервью или RAG. Если они настаивают — отдай bullet points, пусть пишут сами.

---

## Gotcha

SNR — инструмент judgment, не замена ему. Цифры показывают направление, но:

- Если один из интервью-собеседников — ключевой enterprise-клиент, его SNR взлетает ещё выше
- Если RAG нужен другой команде уже на этой неделе — пересчитай Coordination и Leverage вверх
- Пересчитай при изменении данных, не держись за цифры как за истину
