# Skill Disambiguation — какой из похожих выбрать

> Несколько скиллов перекрываются по теме, но различаются по МЕХАНИЗМУ.
> Это не дубликаты — это уровни/режимы. Карта ниже убирает путаницу «их три, какой брать».
> Router/dispatcher и человек используют её, чтобы не вызывать не тот.

## Семья «скепсис / критика»

| Скилл | Механизм | Когда брать |
|-------|----------|-------------|
| **skeptic** | наш Skeptic Engine, 4 режима, тот же контекст | дефолт: red-team идеи/архитектуры/claim **внутри** Claude |
| **codex-skeptic** | запускает **Codex** (другая модель) | когда нужен **кросс-модельный** взгляд — другие слепые зоны (см. doubt-driven-development.md) |
| **grill-me** | интерактивный, вопрос-за-вопросом, ждёт ответа | когда хочешь **диалог** под стресс-тест плана, а не разовый отчёт |

**Правило:** один разовый аудит → `skeptic`. Сомнение в собственной модели → `codex-skeptic`. Живой разбор плана → `grill-me`.

## Семья «исследование»

| Скилл | Механизм | Когда брать |
|-------|----------|-------------|
| **research** | структурный workflow анализа корпуса (fact/inference taxonomy) | разобрать набор статей/доков, что уже есть |
| **research-corpus** | 10-шаговый глубокий corpus-анализ + contradiction/gap finder | когда корпус большой и нужна строгая таксономия |
| **research-scout** | поиск НОВЫХ источников (ArXiv/bioRxiv/Semantic Scholar) | когда надо **найти** статьи, а не разобрать имеющиеся |
| **deep-research** | веб-fan-out + адверсариальная верификация + цитируемый отчёт | многоисточниковый веб-ресёрч с проверкой фактов |
| **research-pipeline** | [EXPERIMENTAL] reference-архитектура multi-agent | не для работы — как образец архитектуры |
| **scientific-research** | научный протокол с гипотезами/эстимандом | строгое научное исследование с FL |

**Правило выбора по вопросу:**
- «найди источники по теме» → `research-scout`
- «разбери эти статьи» → `research` / `research-corpus` (по размеру)
- «что говорят в вебе за месяц» → `last30days`
- «глубокий проверенный отчёт» → `deep-research`
- `research-pipeline` — **не вызывать** как рабочий, это пример

## Семья «гипотезы»

| Скилл | Когда |
|-------|-------|
| **sci-hypothesis** | одна гипотеза, строгая проверка |
| **hypothesis-arbiter** | 4-6 КОНКУРИРУЮЩИХ гипотез, kill-тесты (Chamberlin/Platt) |
| **hypothesis-lab** | панель из 9 «экспертов», многоголосый разбор |
| **consilience** | 5 независимых путей доказательства одной гипотезы |

**Правило:** одна → `sci-hypothesis`; несколько конкурирующих → `hypothesis-arbiter`; нужен «совет» → `hypothesis-lab`; собрать доказательную базу → `consilience`.

## Хуки сессии (для справки — не вызываются вручную)

| Хук | Событие | Роль |
|-----|---------|------|
| `session_save` | по ходу | сохраняет состояние сессии |
| `pre_compact` | перед compaction | спасает важное до сжатия контекста |
| `post_compact` | после compaction | восстанавливает контекст после сжатия |

Не дубли — три разных момента жизненного цикла одной задачи (сохранение / до / после).

---

**Принцип:** если два инструмента дают на одной задаче ОДИН и тот же результат — оставь канонический, второй пометь. Если различаются механизмом (модель, интерактивность, источник) — это не дубль, а режим. Большинство «дублей» в этой системе — режимы.
