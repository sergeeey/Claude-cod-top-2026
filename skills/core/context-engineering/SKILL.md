# Context Engineering — Управление контекстом сессии

## Зачем это нужно

Контекстное окно = рабочий стол. Когда на нём слишком много всего:
- Claude начинает "забывать" детали из начала сессии
- Ответы становятся медленнее и дороже
- Важные решения теряются при compaction

Эта инструкция — система управления "рабочим столом".

---

## Правило 1: Горячее vs Холодное

**Горячий контекст** (держать в окне):
- Текущая задача и её статус
- Последние 3 ошибки и их причины
- Файлы которые редактируются прямо сейчас

**Холодный контекст** (вынести в файлы):
- Архитектурные решения → `.claude/memory/decisions.md`
- Паттерны → `~/.claude/memory/_auto/patterns.md`
- История сессий → `~/.claude/memory/_auto/wiki/`

Если что-то важное — **сохрани в файл, а не держи в промпте**.

---

## Правило 2: Когда делать /compact

Делай `/compact` когда:
- Контекст > 60% заполнен (смотри statusline)
- Закончена логическая единица работы (фича, PR, дебаг сессия)
- Перед сменой задачи

НЕ делай `/compact`:
- В середине дебага (потеряешь стек трейс)
- Если не сохранил activeContext.md

---

## Правило 3: Перед /compact — сохрани

Обязательный чеклист:
```
1. Обновить .claude/memory/activeContext.md (## Current Focus)
2. Если был bug fix → добавить в patterns.md [AVOID]
3. Если было решение → добавить в decisions.md
4. Если длинный дебаг → session_save сохранит в wiki автоматически
```

---

## Правило 4: Структура промптов

**Плохо** (тратит токены на повторение):
```
"Помни что мы используем Python 3.11, Neo4j 5.x, pytest,
и что вчера мы решили использовать asyncpg вместо SQLAlchemy,
и что тесты запускаются через make test..."
```

**Хорошо** (ссылка на файл):
```
"Смотри CLAUDE.md для стека. Задача: [конкретное действие]"
```

Контекст уже в файлах благодаря `session_start.py` и `knowledge_librarian.py`.
Не дублируй его в промптах.

---

## Правило 5: Агенты для длинных задач

Если задача требует >20 tool calls — запускай через агента:
```
Agent(builder) → получает изолированный контекст
```
Это не тратит токены основной сессии на промежуточные шаги.

---

## Диагностика: Признаки переполненного контекста

| Симптом | Причина | Решение |
|---------|---------|---------|
| Claude повторяет вопросы которые уже задавал | Забыл начало сессии | /compact + сохранить контекст |
| Ответы стали короче и поверхностнее | Экономит токены | /compact |
| "Я не нашёл файл X" хотя читал его 5 минут назад | Context eviction | Прочитай файл снова явно |
| Медленная реакция | Большой контекст = больше обработки | /compact |

---

## Метрика

Хорошая сессия: **<40% контекстного окна** на середине задачи.
Если больше → применяй правила 1-3 и делай /compact.
