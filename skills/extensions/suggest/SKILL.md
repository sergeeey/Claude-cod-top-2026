# Suggest — Оркестратор Скилов

Твоя задача: понять что происходит в проекте и рекомендовать 2-3 цепочки скилов
которые дадут максимальный результат для текущей ситуации. Без лишних вопросов.

## Шаг 1 — Собери контекст автоматически

Выполни эти команды и используй вывод для анализа — не копируй результат в финальный ответ:

```bash
# Проект и состояние
pwd
git log --oneline -5 2>/dev/null || echo "no git"
git status --short 2>/dev/null | head -5
git branch --show-current 2>/dev/null
```

Прочитай если существует:
- `.claude/memory/activeContext.md` — текущий фокус
- `CLAUDE.md` — стек и контекст проекта (первые 30 строк)

Если пользователь написал `/suggest <цель>` — это главный сигнал, используй его как основу.

Если нет ни контекста, ни цели (пустой проект / нет git / нет CLAUDE.md) — классифицируй как CLARITY и рекомендуй `/orient → /status` как первую цепочку.

## Шаг 2 — Определи ситуацию

На основе контекста классифицируй ситуацию (одна или несколько):

| Сигнал | Ситуация |
|--------|----------|
| Давно не открывал проект / не помнит контекст | **REENTRY** — возвращение |
| Есть идея / гипотеза для проверки | **HYPOTHESIS** — исследование |
| Нужно принять решение / выбрать направление | **DECISION** — стратегия |
| Готов к релизу / PR / деплою | **SHIP** — выпуск |
| Накопился беспорядок / много задач | **CLARITY** — расчистка |
| Работа с кодом / архитектура / баги | **CODE** — разработка |
| Хочет идеи / inspiration | **CREATIVE** — генерация |
| Карьера / интервью / навыки | **CAREER** — рост |

## Шаг 3 — Подбери цепочки из каталога

Используй каталог ниже. Выбери 2-3 цепочки наиболее подходящие к ситуации.

### Каталог цепочек

**REENTRY — Возвращение в проект**
```
🔄 Быстрый старт:    /orient → /status
   Результат: понимаешь где ты, что сделано, следующий шаг — за 5 минут

🔄 Глубокое погружение:  /orient → /snr → /strategic-analysis
   Результат: полная картина + синтез + стратегический план — за 20 минут
```

**HYPOTHESIS — Проверка идеи**
```
🧪 Быстрая проверка:   /sci-evidence → /skeptic
   Результат: гипотеза сформулирована + атакована — найдены слабые места

🧪 Полный цикл:   /sci-evidence → /skeptic → /validate → /proof-ladder
   Результат: гипотеза прошла фальсификацию → готова к эксперименту
```

**DECISION — Нужно решение**
```
🎯 Синтез → Решение:  /snr → /strategic-analysis → /tracy
   Результат: ключевые инсайты + стратегия + приоритет действий

🎯 Исследование → Решение:  /research → /snr → /strategic-analysis
   Результат: факты + синтез + конкретный план с цифрами
```

**SHIP — Готов к выпуску**
```
🚀 Быстрый релиз:   /orient → /ship
   Результат: release plan с чеклистом за 10 минут

🚀 Качественный релиз:  /tdd-workflow → /security-audit → /ship
   Результат: тесты + безопасность + план релиза — zero сюрпризов
```

**CLARITY — Расчистка хаоса**
```
🧹 Ретро + приоритет:   /session-retrospective → /tracy
   Результат: что сделано → один чёткий приоритет на сегодня

🧹 Полная перезагрузка:  /orient → /tracy → /strategic-analysis
   Результат: контекст → приоритеты → стратегический фокус
```

**CODE — Разработка**
```
💻 TDD цикл:   /tdd-workflow → /security-audit
   Результат: правильные тесты + проверка безопасности

💻 Архитектура:  /brainstorming → /skeptic → /tdd-workflow
   Результат: идеи → атака слабых мест → правильный TDD
```

**CREATIVE — Генерация идей**
```
💡 Идеи → Действие:  /brainstorming → /snr → /tracy
   Результат: много идей → синтез лучших → конкретный приоритет

💡 Нестандартное:  /combinatorial-creativity → /brainstorming
   Результат: неожиданные комбинации → идеи которых не ожидал
```

**CAREER — Рост и интервью**
```
🎓 Момент для обучения:  /career-prep → /grill-me
   Результат: вопрос с интервью из текущей задачи → практика ответа

🎓 Полная подготовка:  /career-prep → /grill-me → /snr
   Результат: вопрос → практика → синтез что улучшить
```

## Шаг 4 — Выдай рекомендацию

Формат вывода — строго одна страница:

```
🎯 SUGGEST — цепочки для [название проекта / ситуации]

📍 Контекст: [1 строка — что происходит]
🏷️ Ситуация: [REENTRY / HYPOTHESIS / DECISION / ...]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔗 1. [НАЗВАНИЕ ПАТТЕРНА]
   Цепочка: /skill1 → /skill2 → /skill3
   Результат: [что получишь — одна строка]
   Когда: [при каком условии эта цепочка лучшая]

🔗 2. [НАЗВАНИЕ ПАТТЕРНА]
   Цепочка: /skill1 → /skill2
   Результат: [что получишь]
   Когда: [условие]

🔗 3. [НАЗВАНИЕ ПАТТЕРНА]  ← быстрый вариант  (можно опустить если только 2 цепочки подходят)
   Цепочка: /skill1
   Результат: [что получишь]
   Когда: [если мало времени]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
▶ МОЙ ВЫБОР: [Цепочка N] — потому что [одна причина]
```

## Правила

- **Не спрашивай** что делать — читай контекст сам
- **Максимум 3 цепочки** — больше = паралич выбора
- **Всегда показывай "Мой выбор"** — одну рекомендацию от себя
- **Одна страница** — никаких длинных объяснений
- Если пользователь написал цель явно (`/suggest проверить гипотезу`) — это главный сигнал
