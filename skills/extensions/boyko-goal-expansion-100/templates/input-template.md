# Input template — GoalContext

Заполняется на Шаге 0 SKILL.md, СНАЧАЛА из контекста разговора и файлов
проекта, не через повторные вопросы к пользователю.

```yaml
GoalContext:
  primary_goal:            # что реально нужно достичь
  success_criteria:        # как измерить, что достигнуто
  target_system:           # объект исследования
  current_state:           # текущая стадия проекта
  known_constraints:       # известные ограничения (ресурсы, время, риск)
  known_attempts:          # что уже пробовали / известные решения
  available_resources:     # ресурсы пользователя (люди, компьют, данные)
  excluded_approaches:     # что пользователь явно хочет исключить
  evidence_standard:       # требуемая строгость доказательств
  time_horizon:            # день / неделя / месяц / год
  risk_tolerance:          # допустимый риск
  requested_count: 100     # по умолчанию 100
  output_language:         # язык ответа
```

Если критически важное поле нельзя вывести из контекста — разрешено задать
НЕ БОЛЕЕ ТРЁХ коротких уточняющих вопросов (не десять, не форму на экран).
