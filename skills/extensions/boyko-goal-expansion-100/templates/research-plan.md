# Research plan — этапы A-F

Внутренний рабочий план скилла (не показывается пользователю как отдельный
артефакт — фиксирует последовательность для самого агента при выполнении
SKILL.md Шагов 2-7).

## Этап A — Формализация задачи
- [ ] Цель переформулирована в проверяемом виде
- [ ] Метрики успеха определены
- [ ] Критерий опровержения всего направления определён
- [ ] Неизвестные переменные выделены
- [ ] Проблема разделена на подзадачи + явные gaps

## Этап B — Базовое исследование
- [ ] Существующие методы найдены (не выдуманы)
- [ ] Недавние исследования проверены (с датой)
- [ ] Конкурирующие подходы найдены
- [ ] Отрицательные результаты / no-go теоремы найдены
- [ ] Аналогичные задачи в других дисциплинах найдены
- [ ] Файлы проекта прочитаны СНАЧАЛА, отдельно от внешних источников

## Этап C — Карта областей
- [ ] Минимум 12 линз выбрано (из references/domain-lenses.md)
- [ ] Каждая линза явно связана с задачей

## Этап D — Генерация пула
- [ ] Внутренний пул ≥150 кандидатов создан
- [ ] Использованы операции генерации (references/domain-lenses.md)
- [ ] Пул НЕ показан пользователю целиком (это сырьё, не результат)

## Этап E — Междисциплинарные комбинации
- [ ] ≥25 вариантов действительно междисциплинарны
- [ ] Каждый прошёл 8 вопросов (references/quality-gates.md)
- [ ] Ни один не основан только на сходстве терминов

## Этап F — Дедупликация
- [ ] Дубли по механизму (не по формулировке) удалены/объединены
- [ ] До 100 сильнейших вариантов выбрано по scoring (references/scoring-model.md)
- [ ] Если <100 содержательных — честно указано фактическое число + причина
