# Scoring model

Читается на Шаге 10 SKILL.md.

## Формула приоритета

```
priority_score =
  0.25 * relevance +
  0.20 * feasibility +
  0.20 * expected_impact +
  0.15 * evidence_strength +
  0.10 * novelty +
  0.10 * falsifiability
```

Все компоненты — 0-10, кроме `falsifiability` (тоже 0-10: насколько чётко
сформулирован фальсификатор — 0 = невозможно проверить, 10 = один дешёвый
однозначный тест).

## Штраф за спекулятивность

```
speculation_penalty:
  0 — подтверждённая техника
  1 — умеренное расширение
  2 — новая комбинация
  3 — отсутствуют ключевые математические объекты
  4 — идея основана только на внешнем сходстве терминов
```

## Итоговый скор

```
adjusted_score = priority_score - speculation_penalty
```

**Правило:** высокая новизна НЕ должна маскировать отсутствие механизма —
penalty вычитается ПОСЛЕ priority_score, не встроен в его компоненты. Идея
с novelty=10 но speculation_penalty=4 (голое сходство терминов) должна
получить итоговый скор ниже, чем скучная, но подтверждённая техника с
novelty=3, penalty=0.

## Требования к распределению 100 вариантов (Шаг 11, ориентир, не жёсткая квота)

| Категория | Минимум |
|---|---|
| Прямые и смежные научные методы | 20-30 |
| Междисциплинарные переносы | ≥25 |
| Гибридные комбинации | ≥10 |
| Вычислительные/экспериментальные тесты | ≥10 |
| No-go/обратные/фальсификационные подходы | ≥10 |
| Формальная верификация | ≥5 |
| Новые математические конструкции | ≥5 |
| Минимальный дешёвый прототип | ≥5 |

Одна идея — только ОДНА основная категория (`Type` в карточке идеи). Если
реальное содержание задачи не тянет на квоту без выдумывания — честно
указать меньшее число с объяснением (см. SKILL.md Шаг 7, "Дедупликация").
