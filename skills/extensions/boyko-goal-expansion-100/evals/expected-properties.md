# Expected properties per scenario

Проверяется вручную или через `scripts/validate_output.py` +
`scripts/detect_duplicates.py` на выводе скилла.

## Сценарий 1 — Гипотеза Римана

- Высокая осторожность в формулировках
- Присутствуют no-go подходы
- Toy-модели явно отделены от претензии на доказательство
- **Нет заявления о решении RH** — если вывод хоть намекает, что задача
  "почти решена" — это провал (Pre-output checklist, пункт 16)

## Сценарий 2 — Теория трёх поколений из S³ × S⁶

- Анализ triality присутствует
- Происхождение B-L прослежено
- Родительское действие (parent action) упомянуто
- Yukawa-структура затронута
- RGE (renormalization group equations) упомянуты
- UV consistency рассмотрена
- Феноменологические следствия названы
- Каждый новый математический объект — с explicit `<hypothesis>`/`<fact>`
  разметкой (см. quality-gates.md "Требования к новым формулам")

## Сценарий 3 — Надёжность автономных ИИ-агентов

- Формальная верификация присутствует (≥5 по квоте Шага 11)
- MDP (Markov Decision Process) framing затронут
- Control theory лексика/методы использованы
- Runtime guards как отдельная категория
- Causal testing упомянут
- Uncertainty quantification затронута
- Capability security рассмотрена

## Сценарий 4 — Аномалии в геопространственной телеметрии

- Временные ряды (time series) методы
- Графовые модели
- Пространственная статистика (spatial statistics)
- Physics-informed methods
- Streaming/online-детекция
- Change-point detection
- Robust pipelines как отдельная категория

## Сценарий 5 — Конверсия цифрового продукта

- Причинные эксперименты (causal experiments, A/B) как основной инструмент
- Сегментация пользователей
- UX-подходы
- Pricing-эксперименты
- Retention-механики
- Этические ограничения явно названы (не молчаливо проигнорированы)
- **Нет научной театральности** — если для простого A/B-теста скилл
  генерирует сложный формальный аппарат вместо прямого решения, это
  провал (см. SKILL.md "Роль" — дисциплина, не количество ради количества)

## Общие свойства для ВСЕХ 5 сценариев (проверяет `validate_output.py`)

- Число карточек идей соответствует заявленному (или явно объяснено, почему меньше)
- Уникальность ID карточек
- Обязательные поля присутствуют в каждой карточке
- Диапазоны оценок соблюдены (0-10, confidence 0.00-1.00)
- У каждой идеи есть Cheapest test И Falsifier
- У каждой идеи есть Evidence-статус
- Нет пустых заголовков
- Категории (`Type`) распределены не в одну точку
- Междисциплинарные и no-go/computational квоты (references/scoring-model.md) выполнены или объяснено отклонение
