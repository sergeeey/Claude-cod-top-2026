# Harvest — Разведка Побочных Активов

## Ядро методологии

**Проблема:** фиксация на исходной цели заставляет выбрасывать алмазы, потому что на них не написано "цель спринта".

**Принцип:** каждый проект = месторождение. Золото — исходная цель. Но рядом всегда есть медь, алмазы, редкие металлы.

**Закон:** не закрывать проект пока не проведена инвентаризация побочных активов.

---

## Типы активов

| Тип | Что искать |
|-----|-----------|
| **code_asset** | скрипты, модули, pipeline, CLI-утилиты, API, парсеры |
| **research_asset** | гипотезы, метрики, модели, наблюдения, классификации |
| **data_asset** | датасеты, логи, разметка, benchmark, телеметрия |
| **process_asset** | промты, workflow, чек-листы, шаблоны, методики |
| **market_asset** | демо, мини-продукт, консалтинговая методика, отчёт |

---

## Режим 1: `/harvest scan [проект]` — быстрая разведка

**Вход:** имя проекта или описание что делалось.

**7 вопросов — задавай по одному, жди ответа:**

```
1. Что была исходная цель проекта?
2. Что реально получилось — совпадает ли с целью?
3. Какие скрипты / модули / функции остались которые работают?
4. Какие неожиданные наблюдения или гипотезы появились в процессе?
5. Какие ошибки повторялись — есть ли паттерн который можно автоматизировать?
6. Что можно переиспользовать в другом проекте прямо сейчас?
7. Что можно показать внешнему человеку (клиенту, партнёру, коллеге)?
```

**Вывод:** список найденных активов с типом → передать в `/harvest score`.

## Режим 2: `/harvest score [актив]` — оценка актива

**Формула Asset Score:**

```
Score = Reuse + Pain + Proof + Uniqueness   (каждый 1–5)
```

| Параметр | Вопрос | 1 | 5 |
|----------|--------|---|---|
| **Reuse** | Можно переиспользовать в другом проекте? | Нет, специфично | Да, универсально |
| **Pain** | Решает реальную боль? | Никто не просил | Люди платят за это |
| **Proof** | Есть работающий код/пример/прототип? | Только идея | Работает в prod |
| **Uniqueness** | Это есть у других? | Банально, всё есть | Твой угол зрения |

**Интерпретация:**

| Score | Действие |
|-------|---------|
| 0–6 | Архивировать — в knowledge base как наблюдение |
| 7–12 | Сохранить как модуль / заметку в Obsidian |
| 13–16 | Выделить в отдельный мини-проект |
| 17–20 | Проверить как продукт / API / исследование / услугу |

## Режим 3: `/harvest full` — полный цикл

Полный деброф всех активных и завершённых проектов.

**Структура вывода:**

```
## Проект: [название]
Исходная цель: [что хотел]
Результат по цели: достигнута / частично / не достигнута

### Найденные активы:
| Актив | Тип | Score | Действие |
|-------|-----|-------|---------|
| [название] | code_asset | 16/20 | мини-проект |
| [название] | research_asset | 11/20 | Obsidian заметка |
| [название] | market_asset | 18/20 | проверить как продукт |

### Топ-1 актив к развитию:
→ [конкретный следующий шаг]
```

---

## Режим 4: `/harvest promote [актив]` — план развития

Когда актив набрал 13+, строим план:

```
Актив: [название]
Тип: [code/research/data/process/market]
Score: [X/20]

Следующий тест (3 дня):
→ [конкретное действие для проверки ценности]

Кому показать в первую очередь:
→ [тип аудитории / конкретный человек]

Kill condition:
→ Если [конкретное условие] — архивируем без guilt.
```

## Ключевой вопрос (запомнить)

> Если исходная цель этого проекта провалилась — **какие активы всё равно остались?**

Это не consolation prize. Это часто и есть настоящий результат.
