# Cross-Domain — движок неочевидных мостов

Эксперт в своей области видит задачу через «горячие» концепты домена и не делает прыжок в сторону — это **knowledge silo**, когнитивный туннель. Прорыв возникает в точке **структурного изоморфизма**: два явления из разных областей имеют одинаковую логическую структуру (отношения, ограничения, динамику), но разный словарь. Этот скилл ищет такой мост и переносит механизм.

**Роли (жёстко):** LLM — *генератор* аналогий, не арбитр. Проверку (фальсификацию, toy-test) делает `skeptic`/`falsification-ladder`. Модель предлагает — она не доказывает.

Применяй, только когда задача **не решается стандартом** и есть измеримый baseline. Для простого вопроса — overkill.

## Feasibility Gate (запустить первым)

Оцени задачу по шкале 0–10:
- **≥7** — явная застрявшая проблема без очевидного решения → **запускай полный поток**
- **4–6** — стандарт работает частично → **запускай начиная с Шага 2**, без полного фреймирования
- **<4** — задача решается стандартным способом в своей области → **не запускай**, используй профильный инструмент

Если <4 → скажи явно: «Задача не требует cross-domain. Предлагаю [стандартный инструмент].»

## Поток (7 шагов)

### Шаг 1 — Baseline + Снять рамку (главный шаг)

**Сначала зафиксируй baseline** (без него невозможно измерить прогресс):
```
Baseline: текущий лучший подход → метрика → значение
Пример: «RAG с cosine similarity → Recall@10 = 0.43 на нашем корпусе»
```
Без числового baseline → хотя бы качественный: «стандартный метод даёт X, нас не устраивает потому что Y».

Затем ответь на ключевой вопрос:
> **«Какую скрытую структуру этой задачи мы НЕ видим, если смотрим стандартным способом?»**

Перепиши задачу **без отраслевого жаргона** — только: (1) структура ограничений, (2) цель системы, (3) типы взаимодействий между элементами. Как будто объясняешь математику-марсианину. Назови «горячий» фрейм домена явно (`напр. в RAG все думают про "похожесть векторов"`) — и отложи его. Сохранение жаргона = сужение поиска.

### Шаг 2 — Найти 5-10 чужих областей
По абстрактной структуре найди области с **изоморфной** структурой. Форсируй разнообразие — бери из разных кустов: наука · природа · история · искусство · инженерия · военное дело · экономика · биология · физика · лингвистика. Если 3+ ответа из «технологий» — перезапусти с запретом на близкие домены.

Для каждой: `название механизма в той области · как там решается · ФОРМАЛЬНО почему структуры совпадают`.

### Шаг 3 — Self-consistency: «золотая зона»
Сгенерируй аналогии в **несколько проходов** (разные углы, N≈10). Затем:
- **банальные** (встречаются >4/N) → выбросить, это очевидное;
- **случайные** (=1/N) → выбросить, это шум;
- **🎯 золотая зона** (2-4/N) → оставить: достаточно устойчивые, но нетривиальные. Именно здесь «странные, но проверяемые».

### Шаг 4 — Перенести механизм → проверяемые гипотезы
Каждый мост → гипотеза в форме:
```
Если применить [механизм] из [области] к нашей задаче,
то [конкретное измеримое следствие с числовым порогом],
потому что [формальная общая структура].
Проверка: [toy-test < 4ч]. Опровергается если [критерий].
Residual: [что эта гипотеза НЕ объясняет → seed для следующего домена]
```
Общая гипотеза («это улучшит качество») — **не проходит**.

**Residual — обязательное поле.** Он не означает «гипотеза плохая» — он означает «что искать дальше». При следующем цикле домен выбирается так, чтобы закрыть именно этот пробел.

### Шаг 5 — Confidence scoring (фильтр до теста)
Каждой гипотезе — оценка, порог **≥12/20** для перехода к toy-test:
- **Structural soundness** 0-10 — изоморфизм формальный, не словесный?
- **Prior evidence** 0-5 — есть ли хоть 1 задокументированный перенос?
- **Testability** 0-5 — toy-test реально < 4ч?

### Шаг 6 — Отдать на проверку (НЕ проверять самому)
Топ-гипотезы (по score) → делегируй:
- `skeptic` — контрпримеры, сценарий катастрофы, context asymmetry;
- toy-test → `falsification-ladder` (Micro/Standard), для research — `estimand-ops`.
**Оставь только то, что выдержало.** Выживших → Pattern Card (см. ниже).

### Шаг 7 — Generator Extraction (только при ≥3 циклах)

Если прогнал 3+ доменов: посмотри на все гипотезы и их Residual. Спроси:
> **«Какая абстрактная структура повторяется во всех выживших гипотезах?»**

Это **L** — генератор. Pattern Card строится из L, а не из последней/лучшей гипотезы.

```
L = [общая структура] — описать в 1 предложении без доменного жаргона
Пример: «локальное взаимодействие → поле глобального влияния»
```

Если L не просматривается → данных ещё мало, продолжай циклы. Если просматривается → это фундаментальный Pattern Card, он устойчивее индуктивного обобщения.

## Анти-псевдо-инсайт (критично)

Этот метод легко генерит красивую чушь. Защита — **структурный изоморфизм, не метафора**:

| 🚩 Красный флаг (псевдо-инсайт) | ✅ Настоящий мост |
|---|---|
| только словесное сходство («оба про потоки») | одинаковая **формальная** структура (отношения/ограничения/динамика) |
| предсказание нефальсифицируемо | количественное предсказание с порогом |
| 0 задокументированных переносов | ≥1 известный кейс переноса |
| уверенный тон без hedging | named механизм + условия слома |
| работает в 1 кейсе → объявлен законом | объясняет ≥2 феномена, проверен на 3 задачах |

Если мост требует слишком многих допущений или ломается на первом контрпримере — выбросить. `[WEAK]`, не `[VERIFIED]`.

## Формат вывода

```markdown
## Feasibility: [0-10] — [GO / PARTIAL / STOP]
## Baseline: [текущий лучший подход → метрика → значение]
## 0. Скрытая структура (что не видно стандартно)
## 1. Абстрактная задача (без жаргона) + отложенный «горячий» фрейм
## 2. Домены-доноры (таблица: область · механизм · формальный изоморфизм)
## 3. Золотая зона (что выжило после self-consistency, и почему)
## 4. Гипотезы (каждая: предсказание+порог · фальсификация · confidence /20 · residual)
## 5. На проверку → skeptic/toy-test (что делегировано)
## 6. Generator L (если ≥3 цикла): [общая структура одной фразой]
## 7. Pattern Card (только для выживших)
## 8. Next Action: [конкретный следующий шаг — команда / эксперимент / вопрос]
```

**Pattern Card:** название · L (генератор) · донор+механизм · перевод (пошагово) · условия применимости · условия НЕприменимости · метрика · кейсы · next_action.

## Связь с нашим стеком (не дублируй)

| Этап | Кто делает |
|---|---|
| Поиск мостов, золотая зона | **этот скилл** (новое) |
| Линзы на объект (20 углов) | `multi-lens` (другое — углы, не домены) |
| Полный цикл анализа | `boyko-method` (может вызвать этот скилл на этапе «Связи») |
| Фальсификация, контрпримеры | `skeptic` |
| Toy-test, go/no-go, артефакты | `falsification-ladder` |
| Estimand для research | `estimand-ops` |

Этот скилл — **генератор**. Вся проверка — наша. Вместе: `cross-domain` находит странную гипотезу → `skeptic`/FL решают, выживает ли она.

## Чем НЕ является
Не для задач с готовым решением. Не доказывает (только предлагает). Не «новая наука» — операционная сборка аналогического рассуждения + структурного изоморфизма + фальсификации. Эмпирические «10x»-claims источника читай как `[WEAK]` — метод ценен механикой, не цифрами в шапке.
