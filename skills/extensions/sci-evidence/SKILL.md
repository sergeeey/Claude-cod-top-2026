# Sci-Evidence — Работа с Гипотезой

## Зачем

Две симметричные ошибки убивают исследования:

1. **Избыточный скептицизм** — застреваешь в фальсификации, никогда не собираешь доказательства
2. **Избыточный оптимизм** — собираешь доказательства не сломав гипотезу, строишь на шатком основании

Этот скил решает обе: сначала ломает (5 миров), потом строит (5 путей). EstimandOps gate защищает от третьей ошибки — нечёткого вопроса.

---

## Шаг 0 — EstimandOps L0 Gate (ОБЯЗАТЕЛЬНО)

**До любой работы с гипотезой — классифицируй тип вопроса.**

| Тип | Форма вопроса | Что оцениваем | Ограничение |
|-----|--------------|---------------|-------------|
| **Descriptive** | "Что такое X в популяции P?" | Сводная статистика | Нет каузальной интерпретации |
| **Predictive** | "Каким будет X для нового случая?" | Условное ожидание | Нет каузальной интерпретации |
| **Causal** | "Что изменится если сделать A?" | Контрфактический контраст | Нужен DAG + проверка идентификации |

**Напиши одно предложение-эстиманд:**
> *"Мы оцениваем [summary measure] [endpoint] для [population], сравнивая [intervention] vs [comparator]."*

**Напиши что результат НЕ означает (минимум 2 пункта):**
- Не доказывает обобщение на [другую популяцию]
- Не устанавливает причинность [если descriptive/predictive]

Эстиманд должен быть написан ДО того как увидел результаты. Если пишешь после — это rationalization.

**Для causal**: нужен DAG + проверка 4 условий идентификации (см. `~/.claude/rules/estimand-ops.md`).

## Режим FALSIFY — "5 Миров Где Я Неправ"

**Цель:** найти 5 независимых способов, которыми гипотеза может быть ложной, ПРЕЖДЕ чем строить на ней.

Детальный протокол для каждого мира → `references/falsify.md`

**Быстрый обзор 5 миров:**

| Мир | Вопрос | Главная угроза |
|-----|--------|----------------|
| 🔴 **Data World** | "Данные врут?" | Biased sample, measurement error, selection bias |
| 🟠 **Method World** | "Метод врёт?" | Confounders, p-hacking, wrong model, circular logic |
| 🟡 **Scope World** | "Эффект слишком узкий?" | Only in this context, doesn't generalize |
| 🔵 **Mechanism World** | "Причина другая?" | Outcome real, but not for claimed reason |
| 🟣 **Replication World** | "Это случайность?" | Effect vanishes on different data or researcher |

**Выход режима FALSIFY:**
- Для каждого мира: вероятность (низкая/средняя/высокая), ключевой тест, kill-условие
- **Вердикт:** ПРОДОЛЖИТЬ / ПИВОТ / ЗАКРЫТЬ
- Если вероятность ≥2 миров = "высокая" → **STOP**, пересмотри гипотезу перед BUILD

## Режим BUILD — "5 Путей к Доказательству"

**Цель:** найти 5 независимых линий доказательства, которые сходятся (consilience). Одна линия — не доказательство. Консилиенс 4+ путей — сильное.

Детальный протокол для каждого пути → `references/build.md`

**Быстрый обзор 5 путей:**

| Путь | Вопрос | Тип доказательства |
|------|--------|-------------------|
| 🔬 **Механистический** | "Почему это должно работать?" | Теория, первые принципы, biochem/math/physics |
| 📊 **Наблюдательный** | "Что говорят данные в природе?" | Корреляции, паттерны, естественные эксперименты |
| 🧪 **Экспериментальный** | "Что происходит при вмешательстве?" | RCT, A/B тест, ablation study |
| 🔁 **Сравнительный** | "Похожие случаи показывают то же?" | Аналогии, cross-domain, animal models |
| 🎯 **Предсказательный** | "Гипотеза предсказала что-то новое?" | Novel predictions confirmed, out-of-sample |

**Выход режима BUILD:**
- Для каждого пути: статус (нет/слабый/умеренный/сильный), что конкретно проверить
- **Consilience Score:** 0–10 (сколько путей и насколько сильно)
- Если Score < 4 → доказательная база недостаточна
- Если Score ≥ 7 → сильная позиция для публикации/решения

## Null Results Protocol

**После режима FALSIFY, если гипотеза отклонена:**

```bash
# Сохранить в null_results/
echo "# Null: [гипотеза]
Дата: $(date +%Y-%m-%d)
Почему отклонена: [мир N, конкретная причина]
Что тестировали: [данные/метод]
Что НЕ стоит повторять: [конкретно]
Что было бы нужно для реабилитации: [конкретные условия]
#null-result #[тема]" > ~/.claude/memory/raw/null-[slug]-$(date +%Y%m%d).md
```

**Почему важно:** Null results пропадают. Через 6 месяцев ты или коллеги повторят ту же гипотезу, потратят время, получат тот же null. Зафиксируй сейчас.

## Gotchas

- **Не пропускай L0 gate** — нечёткий вопрос даёт нечёткие пути доказательства
- **5 миров — не 5 слабостей** — это 5 независимых МИРОВ, каждый с разной логикой фальсификации
- **Consilience ≠ количество статей** — 5 независимых слабых свидетельств хуже 2 сильных независимых
- **Мир #3 (Scope) часто недооценивают** — эффект реален, но только в условиях эксперимента, не generalize
- **Мир #4 (Mechanism) — самый opaque** — outcome может происходить по совсем другой причине
- **Null result ≠ плохой результат** — зафиксируй, не теряй
