# Meeting Insights — Извлечение ценности из встреч

Превращает сырой транскрипт или заметки встречи в структурированные
решения, задачи и знания. Пара с [[SNR Methodology]] — Meeting Gate
решает НУЖНА ли встреча, Meeting Insights — что с неё взять.

## Когда использовать

После любой встречи где:
- Принимались решения
- Назначались задачи
- Обсуждались риски или блокеры
- Нужен протокол для отсутствующих

## Что нужно на входе

- Транскрипт (Whisper, Zoom, Google Meet)
- Ручные заметки в любом формате
- Краткое описание контекста встречи

## Процесс (3 шага)

### Шаг 1 — Подача транскрипта

```
Встреча: [название/тема]
Участники: [список]
Дата: [дата]

Транскрипт/Заметки:
[вставить текст]
```

### Шаг 2 — Извлечение (автоматически)

Промпт для анализа:

```
Проанализируй этот транскрипт встречи.

Извлеки строго в этом формате:

## Контекст
- Встреча: [название]
- Дата: [дата]
- Участники: [список]
- Цель: [одно предложение]

## Решения принятые
- [РЕШЕНИЕ] Что решили, кто решил (owner)

## Action Items
- [ ] [Задача] — Owner: [имя] — Срок: [дата]

## Открытые вопросы
- [ВОПРОС] Что осталось нерешённым

## Риски и блокеры
- [РИСК] Что может помешать

## Ключевые инсайты
- Что важного прозвучало (не задача, но стоит запомнить)

Правила:
- Только то что явно сказано, не додумывай
- Каждый action item = конкретный человек + срок
- Если срок не назван — пиши [СРОК НЕ ОПРЕДЕЛЁН]
- Решения без owner — пиши [OWNER НЕ НАЗНАЧЕН]
```

### Шаг 3 — Сохранение в wiki

Результат автоматически форматируется для сохранения в:
- `~/.claude/memory/raw/meeting-YYYY-MM-DD-название.md`
- При следующей сессии raw→wiki pipeline конвертирует в wiki-запись

## Интеграция с SNR

```
/snr Meeting Gate → встреча одобрена → встреча прошла → /meeting-insights
```

SNR решает: нужна ли встреча.
Meeting Insights решает: что с неё взять.

## Метрики качества встречи (авто-оценка)

После извлечения skill оценивает встречу по SNR-критериям:

| Метрика | Хорошо | Плохо |
|---|---|---|
| Action items с owner | все | часть без owner |
| Решения с rationale | есть | только "решили делать X" |
| Открытые вопросы | <= 2 | > 5 (встреча не завершена) |
| Время на decision | > 50% | < 20% (встреча была информационной) |

Если встреча набирает < 3 баллов → добавляется рекомендация: "следующую встречу заменить async документом".

## Пример вывода

```markdown
## Контекст
- Встреча: Архитектура 24-na-7 core pipeline
- Дата: 2026-04-25
- Участники: Сергей, Claude Code
- Цель: Определить порядок запуска Edge→ASR→Digest

## Решения принятые
- [РЕШЕНИЕ] Начинаем с core pipeline (Edge → ASR → Digest), не с UI — Owner: Сергей

## Action Items
- [ ] Запустить faster-whisper локально и проверить транскрипцию — Owner: Сергей — Срок: 2026-04-28
- [ ] Написать smoke test для ASR модуля — Owner: Claude Code — Срок: следующая сессия

## Открытые вопросы
- [ВОПРОС] Какую модель whisper использовать (tiny/base/large)?

## Риски
- [РИСК] 104 .py файлов никогда не тестировались end-to-end
```

## Gotchas

- Без явного owner action item — мёртвый груз. Настаивай на назначении.
- "Обсудить позже" — не решение. Записывай как открытый вопрос.
- Длинные транскрипты (> 10K слов) — разбивай на части по 30 мин.
