## 1. Оценка потенциала (теоретический потолок)

### A. Рыночная возможность

**TAM/SAM/SOM** — три числа с источниками [VERIFIED/INFERRED]:
- TAM: полный объём рынка
- SAM: достижимый сегмент (с учётом географии, ICP)
- SOM: реалистичная доля за 3 года

Если данных нет → [UNKNOWN], предложи методологию расчёта (bottom-up по ICP × ARPU, не top-down).

**Траектория роста:** CAGR сопоставимых вертикалей [VERIFIED-source].

**Временные факторы:** регуляторные окна, технологические точки перегиба, которые открывают или закрывают окно.

### B. Ресурсная ёмкость

- Техническая масштабируемость: максимальная пропускная способность vs текущая
- Критические пробелы в компетенциях (блокируют экспансию)
- Unit economics в масштабе: LTV/CAC, предельная стоимость на пользователя [INFERRED если нет данных]

### C. Стратегические рычаги

- Неиспользованные потоки выручки (лицензирование, API, данные, white-label)
- Сетевые эффекты: порог критической массы для доминирования
- Защищаемость (Moats): что реально недоступно конкурентам в масштабе

**Итог секции:** ровно 3 числа — "$X ARR потенциал", "Y пользователей потолок", "Z% максимальная доля"

## 2. Текущее использование (фактическое состояние)

### A. Метрики

- **Коммерческие:** выручка, маржа, окупаемость vs бенчмарки [VERIFIED если есть]
- **Операционные:** коэффициенты использования (серверы, команда, adoption фич)
- **Стратегические:** партнёрства, IP, позиция на рынке

### B. Доказательства валидации

- Клиентская: NPS, retention, expansion revenue
- Техническая: uptime, latency, error rate vs SLA
- Рыночная: win rate в конкурентных сделках

**Ограничения:** явно укажи период измерения и качество данных.

## 3. Utilization Ratio

Выбери подход по доступности данных:

**Вариант A (жёсткие данные):**
```
Utilization = Текущая метрика / Максимальная ёмкость × 100%
Пример: $200K ARR / $50M потенциал = 0.4%
```

**Вариант B (качественная оценка):**

| Измерение | Использование | Потенциал | Ratio |
|-----------|--------------|-----------|-------|
| Выручка | [X] | [Y] | [Z%] |
| Рынок | [X] | [Y] | [Z%] |
| Технология | [X] | [Y] | [Z%] |
| Команда | [X] | [Y] | [Z%] |

**Вариант C (pre-revenue):**
Замени деньги на: валидированные гипотезы / всего гипотез, built features / planned features.

**Итог:** Один интегральный Utilization Score (0-100%) с интерпретацией.

## 5. Стратегические рекомендации

Ровно 3 действия с горизонтом:

**Немедленно (0-30 дней):** что разблокирует следующий этап

## Gotchas

- Никогда не опускай Evidence markers ради красоты текста
- TAM top-down (рынок × доля) хуже чем bottom-up (ICP × ARPU × охват) — используй оба
- Если проект pre-revenue → не выдумывай метрики, работай с прокси-показателями
- Kill criteria важнее рекомендаций — без них анализ создаёт false confidence
