# Proof Ladder — Детальный Протокол 7 Уровней

> **Терминологический disclaimer:** Эти 7 уровней = **сила доказательства** (evidence strength).
> Не путать с Falsification Ladder (FL) из `rules/falsification-ladder.md` — там "тиры" (Micro/Standard/Full) = **workflow** для выполнения экспериментов.
> proof-ladder отвечает "насколько убедительно?", FL отвечает "что делать при каждом типе изменений?"

## Уровень 1: Теоретическая Правдоподобность

**Что нужно:** Гипотеза не противоречит известным законам и теориям домена.

**Как проверить:**
- Литературный поиск: есть ли фундаментальные возражения?
- Анализ первых принципов: механизм логически возможен?
- Экспертная консультация: специалист не смеётся при упоминании

**AI-поддержка:** Claude/GPT для поиска теоретических противоречий. Попроси: *"Найди фундаментальные причины почему эта гипотеза не может быть верной по физическим/биологическим/математическим соображениям"*

**Метрика:** Бинарная — нет фундаментальных противоречий.
**Порог успеха:** 0 неразрешимых теоретических возражений.
**Риск:** Гипотеза правдоподобна локально но нарушает что-то в смежном домене.

---

## Уровень 2: Литературная Поддержка

**Что нужно:** ≥3 независимые статьи косвенно поддерживают гипотезу. Нет прямых опровержений.

**Как проверить:**
- PubMed / Google Scholar / Semantic Scholar / arXiv
- Ищи: прямые доказательства + косвенные + смежные домены
- Ищи негативные результаты (PubMed filter: "negative results", "failed replication")
- Ищи **старые термины** — до 2010 использовались другие слова для того же явления

**Старые термины — как искать:**
1. Спроси Claude: "Как называлось [явление] до 2000 года в [домен]?"
2. Посмотри Reference секции первых статей — там старые термины
3. Ищи через citation backward chaining

**AI-поддержка:** Semantic Scholar API, perplexity.ai для быстрого обзора, Claude для синтеза.

**Метрика:** Число статей за/против + качество (impact factor, методология).
**Порог успеха:** ≥3 статьи за, 0 прямых опровержений с сильной методологией.
**Риск:** Publication bias — негативные результаты не публикуются.

**Градация поддержки на этом уровне:**
- 1-2 статьи за: слабая поддержка
- 3-5 статьи, 0 против: умеренная поддержка
- 5+, мета-анализ, 0 прямых опровержений: сильная поддержка

## Уровень 3: Слабые Наблюдательные Данные

**Что нужно:** Паттерн виден в реальных данных без экспериментального контроля.

**Типы данных:**
- Retrospective cohort
- Cross-sectional correlation
- Case series (n≥10)
- Natural experiment (без намеренного вмешательства)
- Реальные данные проекта (логи, метрики, датасеты)

**Как проверить:**
```python
# Минимальный observational check
import scipy.stats as stats
# Pearson/Spearman корреляция
# Mann-Whitney для сравнения групп
# Нет контроля конфаундеров — это OK на этом уровне, но документируй
```

**AI-поддержка:** Exploratory data analysis с Claude, автоматический поиск паттернов.

**Метрика:** p-value + effect size (Cohen's d, r, η²).
**Порог успеха:** p<0.05, ES>0.2 (small effect) в ≥1 датасете.
**Риск:** Confounders — эффект реален, но по другой причине.

**СТОП если:** Baseline model объясняет >90% вариации без твоего механизма → вернись к Шагу 2.

## Уровень 4: Реплицируемый Статистический Сигнал

**Что нужно:** Эффект устойчив в нескольких независимых датасетах или условиях.

**Как проверить:**
- ≥2 независимых датасета показывают тот же эффект
- Cross-validation на held-out data
- Sensitivity analysis (разные модели, разные параметры)
- Subgroup analysis (эффект одинаков в разных подгруппах?)

**AI-поддержка:** Автоматический поиск публичных датасетов (Kaggle, UCI, NCBI GEO, OpenML).

**Метрика:** ES в нескольких датасетах, meta-analytic estimate.
**Порог успеха:** ES>0.3 в ≥2 независимых источниках, CI не включает 0.
**Риск:** Датасеты не по-настоящему независимы (один источник, разные разбивки).

**Что означает этот уровень:** "Слабая поддержка" по шкале публикационной готовности.

---

## Уровень 5: Механистическое Объяснение

**Что нужно:** Есть объяснение ПОЧЕМУ эффект существует, с промежуточными переменными.

**Типы механизмов:**

| Тип | Доказательство | Сила |
|-----|---------------|------|
| Математическая модель | Уравнение предсказывает эффект | Сильный |
| Биохимический путь | Промежуточные молекулы измерены | Сильный |
| Вычислительная симуляция | Агентная модель воспроизводит | Умеренный |
| Аналогия с известным | Похожий механизм в другом домене | Слабый |

**Тест механизма (обязательно):**
> Что случится с Y если заблокировать именно этот механизм, оставив всё остальное?
> Если Y не меняется → механизм неверный.

**AI-поддержка:** Claude для поиска аналогов механизма в других доменах, симуляции.

**Метрика:** Промежуточные переменные измерены + медиационный анализ.
**Порог успеха:** Медиатор объясняет ≥50% эффекта (Baron-Kenny или SEM).
**Риск:** Ad hoc explanation — механизм придуман после наблюдения (не фальсифицируемый).

## Уровень 6: Контролируемый Эксперимент

**Что нужно:** Эффект при намеренном вмешательстве с контрольной группой.

**Типы (по силе):**

| Тип | Условия | Сила |
|-----|---------|------|
| Pre-registered RCT (double-blind) | Рандомизация + ослепление | Очень сильный |
| RCT (open label) | Только рандомизация | Сильный |
| Quasi-experiment | Частичный контроль | Умеренный |
| A/B тест (production) | Рандомизация по юзерам | Сильный в домене |
| Ablation study (ML) | Убираем компонент | Умеренный |

**Pre-registration обязателен** (OSF, AsPredicted, ClinicalTrials). Без pre-reg → уровень снижается до 4.

**AI-поддержка:** Power analysis (G*Power), рандомизация, автоматический мониторинг.

**Метрика:** Primary outcome (pre-registered) + ITT analysis.
**Порог успеха:** p<0.05 на primary outcome, ES в ожидаемом диапазоне.
**Градация:** "Умеренная-сильная поддержка" / "Предрегистрационно готово".

## Уровень 7: Независимая Репликация

**Что нужно:** Другая команда, другие данные, тот же результат.

**Что считается репликацией:**
- Другая лаборатория / команда (не аффилированная)
- Другой датасет из другого источника
- Другой метод который приходит к тому же выводу (консилиенс)
- Прямая репликация протокола уровня 6

**AI-поддержка:** Поиск попыток репликации (Replication Wiki, PsychFileDrawer, поиск "failed replication + [тема]").

**Метрика:** ≥1 успешная независимая репликация + отсутствие прямых опровержений.
**Порог успеха:** ≥1 прямая репликация OR ≥2 независимых команды нашли тот же эффект.
**Градация:** "Публикационно готово" (1 репликация) → "Требует широкой репликации" (>5 команд).

**Что значит FAIL на этом уровне:**
- Не паника. Pre-registration + full disclosure о non-replication — правильный научный путь.
- Запусти 5 миров FALSIFY заново: что изменилось?
- Проверь: scope issue (Мир 3) или method (Мир 2)?
