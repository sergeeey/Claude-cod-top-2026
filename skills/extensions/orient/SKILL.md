# Orient — Project Briefing

Твоя задача: дать пользователю одностраничный брифинг проекта за минимальное время.
Пользователь открыл папку которую не видел несколько дней/недель и хочет вспомнить
где он был и что делал дальше.

## Шаг 1 — Определи папку проекта

Если пользователь указал путь в сообщении — используй его.
Иначе — текущий рабочий каталог (`.`).

## Шаг 2 — Собери контекст (читай параллельно, всё что найдёшь)

Читай в порядке приоритета — каждый файл добавляет слой понимания:

1. **`CLAUDE.md`** — цель проекта, стек, ключевые решения (главный источник)
2. **`README.md`** или **`README.rst`** — публичное описание проекта
3. **`.claude/memory/activeContext.md`** — где остановились, что в процессе
4. **`git log --oneline -7`** — последние коммиты (что реально менялось)
5. **`git branch --show-current`** — текущая ветка
6. Любые файлы с именами `*TODO*`, `*PLAN*`, `*BACKLOG*`, `*activeContext*` в корне или `.claude/`

Если файл не найден — пропускай без ошибки. Работай с тем что есть.

## Шаг 3 — Выведи брифинг

Используй этот формат (пропускай секции если данных нет):

```
🗺️ ORIENT — [Название проекта]
════════════════════════════════

📌 ЧТО ЭТО
  [1-3 строки: цель проекта и для кого]
  Стек: [технологии через запятую]

📅 ПОСЛЕДНЯЯ СЕССИЯ ([N дней назад / дата последнего коммита])
  • [последний коммит или действие — конкретно]
  • [ещё 1-2 ключевых факта из activeContext или git log]
  Остановились на: [конкретная точка остановки если есть]

⏳ ЧТО ОСТАЛОСЬ
  • [пункт из TODO/activeContext/PLAN]
  • [ещё пункты, максимум 5]

🔴 ГОРИТ (только если есть реально критичное)
  • [дедлайн, сломанный CI, блокер]

▶ СЛЕДУЮЩИЙ ШАГ
  [Одно конкретное действие чтобы продолжить работу]
```

## Правила качества

**Конкретно, не абстрактно.** Вместо "работали над кодом" → "починили баг в auth middleware (коммит a3f12c)".

**Честно при отсутствии данных.** Если нет activeContext и нет TODO — так и скажи:
"Нет файлов контекста. Вот что вижу в git: [log]". Не придумывай.

**Коротко.** Брифинг должен умещаться на один экран. Если данных много — выбирай самое важное.

**Без вопросов.** Пользователь хочет быстрый ответ, не диалог. Собери всё сам и выдай результат.
Единственное исключение: если совсем непонятно какой проект имеется в виду — спроси путь.
