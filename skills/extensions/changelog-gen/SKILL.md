# Changelog Generator

Превращает технические git коммиты в polished release notes которые
пользователи реально прочитают.

## Когда использовать

- Подготовка релиза новой версии (перед `git tag`)
- Еженедельный/месячный дайджест изменений
- Release notes для GitHub Releases
- Changelog для CHANGELOG.md

## Процесс (4 шага)

### Шаг 1 — Сбор коммитов

```
git log --oneline --no-merges v{prev}..v{next}
```

Или за период:
```
git log --oneline --no-merges --since="7 days ago"
```

### Шаг 2 — Категоризация

| Категория | Коммиты | Emoji |
|---|---|---|
| Новые фичи | feat: | ✨ |
| Улучшения | perf:, refactor: | 🔧 |
| Баги | fix: | 🐛 |
| Безопасность | security:, sec: | 🔒 |
| Breaking changes | BREAKING: | ⚠️ |

Фильтровать (не включать в user-facing): `test:`, `chore:`, `ci:`, `docs:` — внутренние.

### Шаг 3 — Перевод технического → понятного

| Технический коммит | User-facing |
|---|---|
| fix: null pointer in auth middleware | Исправлена ошибка при входе на отдельных устройствах |
| feat: add isotonic calibration | Улучшена точность предсказаний |
| perf: batch processing 3x speedup | Обработка данных стала в 3 раза быстрее |

### Шаг 4 — Форматирование

```markdown
# Что нового — v{version} ({date})

## ✨ Новые возможности
- **Название фичи:** Одно предложение что это даёт пользователю.

## 🔧 Улучшения
- Краткое описание улучшения без технических деталей

## 🐛 Исправления
- Что было сломано и как теперь работает

## ⚠️ Важные изменения
- Если что-то изменилось в поведении — объяснить что сделать пользователю
```

## Промпт для быстрого запуска

```
Сгенерируй user-facing changelog из git log за последние 7 дней.
Правила:
- Исключи: test:, chore:, ci:, docs: коммиты
- Переведи технический язык в понятный пользователю
- Сгруппируй по категориям: фичи / улучшения / фиксы
- Формат: markdown с emoji
- Язык: русский
```

## Пример вывода

```markdown
# Обновление — 25 апреля 2026

## ✨ Новые возможности
- **SNR-скоринг задач:** Новый инструмент /snr оценивает задачи
  по формуле Signal/Noise и подсказывает что удалить из бэклога.

## 🔧 Улучшения
- Wiki-записи теперь автоматически проверяются на overgeneralization
  — вы увидите предупреждение если вывод слишком категоричный.

## 🐛 Исправления
- Исправлена ошибка при конвертации заметок с процентными значениями.
```

## Интеграция с /ship

Этот skill вызывается автоматически внутри `/ship` перед созданием git tag.
Можно использовать standalone для промежуточных дайджестов.

## Gotchas

- Коммиты без префикса (feat/fix/etc) — решай по смыслу, не игнорируй
- Breaking changes всегда наверху, даже если их мало
- Не переусердствуй с emoji — один на категорию, не на каждую строку
